Object-Oriented Programming

Sprint 1: Getting Started, Object-Oriented Programming Concepts, and Tools of the Trade


Welcome to Object-Oriented Programming! By this point in your programming journey I'm sure that you are confident in writing basic software programs. You've learned about structured programming techniques like sequence, selection, and repetition. You've learned how to break complicated programs down into more manageable pieces by using functions. That's great, but to create modern programs you need to master some additional skills. These additional skills are what object-oriented programming give us.

You have likely noticed that I use the term “sprint 1” to describe our first set of course activities and assignments. Sprints are part of the Scrum development process. The term is used to describe a unit of work which will be delivered in a given amount of time. We will be learning more about sprints and Scrum in the coming days. For now it is sufficient to know that our sprint 1 activies listed below are expected to take 8 to 12 hours of focused effort to complete.

Let's start with what we hope to learn in sprint 1. By the end of sprint 1 we will be able to:

  • Identify and understand key object-oriented programming concepts and practices
  • Access course material utilizing Blackboard and the GitHub
  • Utilize command line and graphical tools to navigate our computer operating system and local file system
  • Create, edit, and update source code in a text editor
  • Compile and execute basic Java applications utilizing Java command line tools
  • Manage source code utilizing the GitHub and the Git command line client
